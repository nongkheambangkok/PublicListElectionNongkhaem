ไฟล์นี้ออกแบบมาให้ใช้งานง่ายบน GitHub Pages โดยเน้นความปลอดภัยของข้อมูลส่วนบุคคล (PDPA)

1. การเตรียมข้อมูล (สำหรับ Admin)

เนื่องจากเราไม่ควรนำไฟล์ CSV ดิบที่มีเลขบัตรและชื่อจริงขึ้น GitHub ตรงๆ ให้ทำดังนี้:

เปิดไฟล์ index.html ขึ้นมาบน Browser

คลิกที่แท็บ "สำหรับผู้ดูแล (สร้างข้อมูล)"

เตรียมข้อมูล CSV โดยเรียง Column ดังนี้:
ชุดที่, จังหวัด/เขตเลือกตั้ง, ลำดับที่, เลขบัตรประชาชน, ชื่อตัว-ชื่อสกุล
(ไม่ต้องใส่ Header แถวแรก)

Copy ข้อมูล CSV มาวางในช่องรับข้อมูล

กดปุ่ม "แปลงและเข้ารหัสข้อมูล"

ระบบจะสร้าง JSON Code ชุดใหม่ขึ้นมา ให้กดปุ่ม Copy

Save ไฟล์ และ Upload ขึ้น GitHub ได้เลย

3. ความปลอดภัย (Technical Detail)

เลขบัตรประชาชน: ถูก Hash ด้วย SHA-256 ทำให้คนเห็น Code ไม่สามารถแปลงกลับเป็นเลขบัตรจริงได้ (One-way)

ชื่อ-สกุล: ถูก Encrypt ด้วย AES โดยใช้ เลขบัตรประชาชนเป็น Password

ถ้า Hacker ได้ไฟล์ไป แต่ไม่รู้เลขบัตร ก็จะถอดรหัสชื่อไม่ได้
